# AP1-Konfigurator v1.0.11

## Highlights

- Die Projektdokumentation folgt jetzt dem gleichen `docs/`-Muster wie die Schwesterprojekte.
- Der aktuelle Release-Stand ist damit sowohl lokal als auch auf GitHub wieder vorhanden.
- Anwender-, Technik-, Kurz- und Checklisten-Dokumentation sind nun sauber getrennt.

## Änderungen im Detail

### Dokumentationsstruktur

- neue kanonische Dateien unter `docs/` angelegt:
  - `docs/DOKUMENTATION_ANWENDER.md`
  - `docs/DOKUMENTATION_TECHNIK.md`
  - `docs/KURZDOKUMENTATION.txt`
  - `docs/DOKUMENTATION_CHECKLISTE.md`
- `DOCUMENTATION.md` dient nun als Weiterleitung auf die technische Doku im `docs/`-Ordner.
- `docs/Liesmich.txt` dient nun als Weiterleitung auf `docs/KURZDOKUMENTATION.txt`.
- `README.md` verweist auf die neuen kanonischen Dokumente.

### Release

- Versionsstand auf `v1.0.11` angehoben.
- Release-Unterlagen für den aktuellen Stand ergänzt.
- Lokales Release-Paket und GitHub-Release werden für denselben Versionsstand bereitgestellt.

## Validierung

- Fehlerprüfung der geänderten Markdown- und Textdateien ohne Befund.
- Dokumentationsverweise im Projektwurzelverzeichnis auf das neue `docs/`-Schema umgestellt.
